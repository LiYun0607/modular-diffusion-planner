Abstract

Bridging the gap between benchmark performance and vehicle-deployable performance.

Diffusion-based motion planners have achieved state-of-the-art results on benchmarks such as nuPlan, yet their evaluation within closed-loop production autonomous driving stacks remains largely unexplored. Existing evaluations abstract away ROS 2 communication latency and real-time scheduling constraints, while monolithic ONNX deployment freezes all solver parameters at export time.


We present an open-source modular benchmark that addresses both gaps: using ONNX GraphSurgeon, we decompose a monolithic 18,398-node diffusion planner into three independently executable modules and reimplement the DPM-Solver++ denoising loop in native C++. Integrated as a ROS 2 node within Autoware, the open-source AD stack deployed on real vehicles worldwide, the system enables runtime-configurable solver parameters without model recompilation and per-step observability of the denoising process.


Through systematic comparison of DPM-Solver++ (first- and second-order) and DDIM across six step-count configurations (\(N \in \{3, 5, 7, 10, 15, 20\}\)), we show that encoder caching yields a 3.2× latency reduction, and that second-order solving reduces FDE by 41% at \(N{=}3\) compared to first-order.

Getting Started

Run the modular diffusion planner in Autoware + AWSIM.

1
Prerequisites

The following software environment is required:

# System
Ubuntu 22.04 LTS (Jammy Jellyfish)
ROS 2 Humble
Autoware (latest)
NVIDIA GPU with CUDA support

# Dependencies
ONNX Runtime 1.23+
AWSIM (for closed-loop simulation)
Nishishinjuku HD Map
2
Docker (Recommended)

A pre-built Docker image with all dependencies will be available upon paper acceptance.

# Coming soon
docker pull <image-name>  # TBD
docker run --gpus all -it <image-name>
3
Build from Source

Build the modular planner node and the planning launch configuration:

cd ~/autoware_ws

# Source Autoware and workspace
source /opt/autoware/setup.bash
source install/setup.bash

# Build the modular planner and planning launch
colcon build \
  --packages-select autoware_diffusion_planner_onnx_split_cpp \
                    tier4_planning_launch \
  --cmake-args -DENABLE_ROS_NODE=ON
4
Environment Setup

Configure ONNX Runtime library paths before launching:

# Set up ONNX Runtime environment variables
source ~/autoware_ws/setup_onnx_env.sh

# Source Autoware and workspace
source /opt/autoware/setup.bash
source ~/autoware_ws/install/setup.bash
5
Launch Closed-Loop Simulation

Start the full Autoware stack with AWSIM on the Nishishinjuku urban route:

ros2 launch autoware_launch e2e_simulator.launch.xml \
  vehicle_model:=awsim_labs_vehicle \
  sensor_model:=awsim_labs_sensor_kit \
  map_path:=/autoware_map/nishishinjuku_autoware_map \
  launch_sensing_driver:=false
6
Runtime Configuration

All solver parameters are exposed as ROS 2 parameters and can be changed at runtime without restarting:

# Change denoising step count
ros2 param set /diffusion_planner num_steps 5

# Switch between 1st-order and 2nd-order solver
ros2 param set /diffusion_planner solver_order 2

# Adjust VP noise schedule
ros2 param set /diffusion_planner beta_start 0.1
ros2 param set /diffusion_planner beta_end 20.0
